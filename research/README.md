# /research

Discovery + market work that happens BEFORE any code is written. Markdown only — no PDFs, no DOCX. Plain text means future contributors (human or Claude) can challenge or update assumptions without losing the trail.

## Standard files

- **`ICP.md`** — five customer personas: trigger-fear, desired outcome, decision criteria, voice, page assignment, lead magnet. Locked in Week 1; quarterly review thereafter.
- **`BRAND.md`** — brand archetype, voice rules, the founder's three operating rules, anti-patterns. The spine of every customer-facing decision.
- **`MARKET.md`** — competitive landscape, pricing, positioning moat. What makes this offer defensible.
- **`KEYWORDS.md`** — SEO keyword research, search-volume estimates, long-tail clusters.
- **`MAP-PACK.md`** — local-SEO plan: city target list, Google Business Profile setup, NAP-consistency audit, internal linking pattern.
- **`INTERVIEW-NOTES.md`** — raw discovery interviews with the founder. Quote freely — these become source material for blog posts.
- **`SEO-DASHBOARD.md`** — monthly SEO + map-pack metrics review (after launch).

## Optional files

- **`NAP-CITATIONS.md`** — per-citation status across the directory properties (GBP, Yelp, Apple Maps, etc.)
- **`MAP-PACK-CITIES.md`** — per-city status: page built, GBP citation, blog post coverage
- **`AUDIT.md`** — pre-rebuild legacy site audit, if migrating from an existing site

The `/research/legacy-site/` subfolder (if present) holds the captured extracts from the prior site (raw HTML, content JSON, design tokens) for reference during the rebuild.
