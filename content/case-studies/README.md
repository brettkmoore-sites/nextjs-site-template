# /content/case-studies

Long-form proof stories — different from `/content/blog/` because they're explicitly framed as proof artifacts (named client + specific dollar/MPH/outcome + step-by-step process). Often imported as evidence into multiple pages.

## Frontmatter

```yaml
---
client_name: "<First Last>"
title: "<Specific outcome>: <plain-English what this is>"
slug: <client-or-outcome-slug>
date: 2026-04-12
outcome_metric: "$250,000 over asking in 5 days"
hero_image: /images/case-studies/<slug>-hero.jpg
---
```

Body follows the case-study blog pattern (Pattern 5 from the playbook): The Property/Setup → The Counter-Intuitive Answer → The Plan (numbered steps) → The Outcome → The Math → The Lesson.
